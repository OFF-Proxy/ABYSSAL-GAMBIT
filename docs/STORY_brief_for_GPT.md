# ストーリー作成のためのゲーム仕様ブリーフィング（ChatGPT向け）

このドキュメントは、シナリオ担当（ChatGPT）が **ゲームシステムとマッチした物語** を書けるように、
現状の仕様・物語に関わる仕組み・今後の機能を一枚にまとめたものです。
（世界観/プロット案は別紙 `DESIGN_story.md` を参照。本書はそれを“どう物語として届けるか”の制約と素材集です。）

---

## 1. ゲームの基本（これが物語の“器”）
- ジャンル: **2Dローグライク・オートチェス／ボスラッシュ**。
- 1プレイ(ラン)の流れ: **章を1つ選ぶ → 複数ウェーブを連戦 → 最後に章ボス**。各ウェーブの間に「編成フェーズ」がある。
- 編成フェーズでやること: ショップでユニット購入 → 盤面に配置 → シナジー/星上げ/アイテム/オーグメントで強化 → 戦闘は**オートで進行**（プレイヤーは編成と必殺発動が主）。
- **主人公＝ヒーロー1体を選んでランを始める**。ヒーローは盤面枠を消費しない特別枠で、固有の**必殺スキル**を持つ。
- 物語は「自由に歩き回るRPG」ではなく、**章＝ボスへの道**という構造。だから**物語は短いビートの積み重ね**で届ける必要がある（後述の会話システム）。

### 物語の“尺”の制約（重要）
- 現在ストーリーを語る主な場所は **ボス戦前の掛け合いダイアログ**（VN風・全画面）。
- 1回のダイアログは原則 **3行**:
  1. ボスの挑発
  2. ヒーローの返し
  3. ボスの締め（→戦闘開始）
- つまり**「3行で1シーン」**が基本単位。長い説明より、キャラの立つ短い応酬が向く。
- 章クリア時にボスが仲間化する（後述）ので、「倒した相手の事情を知り、力を継ぐ」という物語付けが自然。

## 2. 物語に直結するゲームシステム（シナリオが守るべき/活かせる点）

### (A) ヒーロー＝視点キャラ（主人公は選択式）
- プレイヤーはランごとに主人公ヒーローを選ぶ。→ **物語は「各ヒーロー視点」で成立しつつ、最終的に収束**してほしい。
- ヒーローは**進行度で解放**される（下記表）。→ 物語上の“登場/合流”を解放章に合わせると自然。

### (B) 仲間化（リクルート）
- **章ボスを倒すと、そのボスが恒久的に使えるユニットとして解放**（仲間化）。中ボスも一時解放あり。
- 物語的意味づけ: 「敵将を打ち破り、その力（と事情）を受け継ぐ」。災厄陣営の将が味方になっていく＝終焉に抗う側が増える構図に使える。

### (C) 熟練度(Mastery)
- 選んだ主人公でチャプターをクリアすると**熟練度**が上がる（最大Lv50・逓増）。
- Lvが上がると、そのヒーローの**必殺スキルのバリアント（強化A/B）が解放**され、カスタムできる。
- 物語フック: 「使い込むほどヒーローの“在り方”が深まる」。ヒーロー別アークの掘り下げと相性が良い。

### (D) シナジー（陣営＝物語の勢力そのもの）
- 6陣営シナジー: **ライアナー/ソンガイ/ヴェツルヴィアン/アビサル/マグマー/ヴァナー**（=物語の6勢力と一致）。
- 加えてテーマ系シナジー（戦士/狙撃/秘術/守護/獣/影/機巧/幽鬼/原典/業火/氷霜/嵐/深淵/聖光/狂乱/王権/召喚/錬金 等）。
- 陣営シナジーは **1/3/5/7/10体** で段階発動。**10体は「ほぼ確定クリア」級**の特大効果。
- 物語フック: 「同じ陣営で固めると“その勢力の総力”が出る」＝陣営の結束/思想を戦術で体感できる。

### (E) スキン（姿替え＝物語イベントの報酬）
- **Skindogehai＝カガチの“犬の姿”**。設定: 章8でアビサルのキャシヴァに犬に変えられる。**章8をカガチでクリアすると、以後カガチを犬の姿(スキン)で遊べる**ように解放。
- → 物語イベントが“見た目報酬”として残る仕組み。今後ほかのヒーローにも展開余地。

### (F) ロビー（ハブ）/図鑑
- セーブスロット選択 → メインロビー（ハブ）→ モード選択 → 章選択 → 出撃。
- ハブに「ヒーロー育成（熟練度・必殺カスタム）」「図鑑（ボス/ユニット）」がある。
- → 将来「ストーリー/回想ビューア」をハブに置く余地あり（後述・今後の機能）。

## 4. シナリオに守ってほしい制約（ゲーム性とのマッチ）
1. **3行1シーン**で成立すること（ボス戦前ダイアログ＝主要な語り口）。長尺は幕間枠ができてから。
2. **各ヒーロー視点で成立しつつ、最終章(アルカナ)で収束**。主人公はランごとに選択式＝「誰で進めても物語が通る」必要。
3. **進行＝解放**。ヒーロー/ボス将は章クリアで増える。物語上の登場/合流を解放章に合わせる。
4. **倒した敵が味方になる**(仲間化)。敵将を“理解して受け継ぐ”筋が自然。最終的に元・災厄が終焉に反旗、が映える。
5. **陣営＝勢力**。陣営シナジーで固める＝その勢力の思想/結束を体感。陣営の性格づけを一貫させる。
6. **ボスラッシュ構造**（自由探索なし）。世界の説明は「戦いの前後の短い応酬」と「図鑑/回想」で小出しに。
7. 既存トーン参考（実装済みの仮セリフ例）:
   - アルカナ「全ての陣営を従えし我こそが終焉。ここがお前の旅の果てだ。」
   - キャリバー「我はキャリバー・O。光の名のもとに、貴様の進軍はここで断つ。」
   - ヴァース「原始の炎よ、燃え盛れ。貴様の軍勢、灰にしてくれる。」

## 5. 今後の機能（物語が見据えるべき拡張余地）
- **幕間ダイアログ枠**: 章の合間/合流イベント用に、ボス戦前以外でもVN会話を出せる枠を追加予定。→ 中盤の合流シーン等を“3行縛り”を超えて描ける。
- **ストーリー/回想ビューア（ハブ）**: 解放済みの会話/イベントを後から読める画面。→ 小出しにした断片を回収できる。
- **ヒーロー別の必殺A/Bバリアント**: 熟練度で分岐。各ヒーローの“成長/選択”を物語と連動させられる。
- **スキン拡張**: 犬化カガチ(Skindogehai)同様、物語イベント→姿替え報酬を他ヒーローにも。
- **掛け合いの拡充**: 現状はボス別/ヒーロー別＋汎用フォールバックの3行。理想は **「各ボス × 各主人公」専用の3行**（最低でも陣営×ヒーロー単位）。
- **立ち絵**: ヒーローは一枚絵あり(HeroArt)。ボスは専用立ち絵が一部のみ→今後追加余地（会話の演出強化）。

## 6. ChatGPTへの“成果物オーダー”例（こう出してくれると実装しやすい）
シナリオを次の形で書いてもらえると、そのままゲームへ流し込めます。

1. **章あらすじ（全13章）**: 各章 3〜6行。誰が動き、何と対峙し、何が明かされるか。合流イベント(中盤)もここに明記。
2. **ボス戦前ダイアログ**: 「**ボスID × 主人公ヒーローID**」ごとに3行（ボス挑発／ヒーロー返し／ボス締め）。
   - 最低限: 陣営/ボス別＋主人公の基本3体(アルディン/カガチ/ヴェスナ)分。理想: 全主人公×全ボス。
   - 例フォーマット:
     ```
     [boss=Abyssalcassyva][hero=HeroKagachi]
     1(boss): …（キャシヴァの挑発。カガチを犬に変える示唆）
     2(hero): …（カガチの返し）
     3(boss): …（締め→戦闘）
     ```
3. **特殊イベント**: カガチ犬化(章8)、第一合流(章4-6)、最終収束(章13)のシーン台本（幕間枠想定で少し長くてOK）。
4. **各ヒーローの一行コンセプト＆最終章での“答え”**（収束の核）。
5. **各ボス将の一行コンセプト**（動機の核）。

> 上記が揃えば、こちらで `HeroBossDialogueUI`／幕間枠／図鑑紹介文へ実装します。
> 表記は **日本語が主**、英語は任意（実装は JA/EN 両対応）。
