## 0. 物語の核（STORY_MASTER.md より）

> テーマ:「終焉に抗う物語」ではなく **「世界を諦めた観測者（アルカナ）に、もう一度未来を信じさせる物語」**

キーワード: **希望 / 諦め / 未来 / 選択 / 仲間 / 理解**。ラスボス＝アルカナ（最終章 ch20）。
主人公はプレイヤーが選んだヒーロー。物語は「世界共通の筋＋主人公ごとの視点差分」で構成する。

### 私（ユーザー）がどう作りたいか（=創作意図）

既存セリフから読み取れる、踏襲してほしい流儀：

- **ボスは「思想」を体現する**。各ボスは1つの概念（後悔・覚悟・本能・別れ・救済・管理・諦め・運命…）を背負い、
  その思想で主人公を否定・誘惑してくる。倒すこと＝その思想への主人公なりの「答え」を示すこと。
- **主人公は「核となる価値観」で返す**。各ヒーローは一貫した信条を持ち、どのボスにも自分の言葉で応える
  （例：アルディン＝「守れぬものを抱えたまま守り続ける」、カガチ＝「弱くても進む」、ヴェスナ＝「居場所は作れる」）。
- **説教くさくしない**。短く、ぶっきらぼうでも芯のある言い回し。叙情に流れすぎず、戦う直前の緊張感を保つ。
- **アルカナだけは特別**。覇王口調ではなく **ですます＋いたわり**。敵意ではなく「もう休んでいい」と諭す慈愛。
  だから倒すのではなく「その先を見せて翻意させる」決着になる。
- **犬化（カガチ ch8）はギャグでなく再起の物語**。屈辱→生きる実感→「この姿でも進む」。キャシヴァは生の肯定者。

> ①②③の本文は **日本語確定稿のみ使用**。英語は当面 ④ の汎用英語へフォールバック（EN は今回の発注対象外）。

### 行数とパターン（重要）

- ①②③とも **可変行数**。配列の **偶数index(0,2,4…)＝ボス側 / 奇数index(1,3,5…)＝主人公側**（ボスで開始・交互）。
  3行に限定しない。①章ボス戦前は **6〜9行**推奨、②中ボスは **3〜5行**、③撃破後は **2〜4行**（短い余韻）。
- 立ち絵の左右・位置はシステムが自動。台本に位置指定や話者タグは不要（**交互の順番だけ守る**）。

## 6. 出力フォーマット（Codex → Cowork へ渡す形）

そのままC#へ移せるよう、**ブロック単位**で。交互順（ボス開始）を厳守。

### ① 章ボス戦前 / ③ 撃破後
```
[prefight boss=Caliber hero=HeroAldin]   ← ③は prefight を postboss に
1(ボス): …
2(主人公): …
3(ボス): …
4(主人公): …
5(ボス): …
6(主人公): …
（戦前は6-9行 / 撃破後は2-4行）
```

### ② 中ボス（主人公共通）
```
[midboss boss=Silitharelder]
1(ボス): …
2(主人公): …
3(ボス): …
（3-5行 / 主人公は「共通の言い回し」で可。固有名は避ける）
```

- bossId / heroId は §2・§3 の表記（大文字小文字そのまま）。Cowork がキーを小文字化して登録する。
- 1ブロック＝1組。**組の取りこぼしが無いように**、対象マトリクス（§7）のチェックリストを埋める形で出す。
- JA確定稿のみでよい（EN不要）。
