# 07 Dialogue Guide

## 基本ルール

ボス戦前は原則3行です。

```text
1. ボス: 思想または挑発
2. 主人公: その主人公の答え
3. ボス: 戦闘開始への締め
```

長尺の説明は幕間、観測片、図鑑へ逃がします。  
戦前3行は「思想の衝突」を見せる場所です。

## アルカナの絶対ルール

- ですます調。
- 怒鳴らない。
- 悪役口調禁止。
- 「滅ぼす」より「休ませる」「眠らせる」「もう十分です」。
- 相手を憎んでいない。
- 自分が完全に間違っていたとは最後まで思わない。

例:

```text
まだ歩くのですね。
疲れませんか。
もう休んでもいいのですよ。
続きを……観てみたいですね。
```

アルカナの個人的な喪失は `12_Arcana_Private_History.md` を正本とします。  
ネリ、オルハ、白紙戦争は強い材料ですが、説明しすぎると台詞が重くなります。戦闘中は短い断片として出し、詳細は観測片とライブラリで補完します。

## 中ボス

中ボスは物語の核心を語りすぎない。  
役割は章ボスの思想を予告することです。

- 軽い挑発
- 世界観補足
- コメディ可
- 章ボスより短く
- 同じ素体でも章ごとに別個体扱い可

## 覚醒イベント

覚醒台詞は、勝利宣言ではなく「弱さの扱い方が変わる瞬間」として書きます。  
覚醒設計、章配置、素材対応は `15_Hero_Awakening_Design.md` を正本とします。

書き方:

- 敵や章テーマが主人公の弱さを突く。
- 主人公は弱さを否定しない。
- 「だから強い」ではなく「弱いままでも選べる」と返す。
- 覚醒後も一人称と口調を変えない。
- 覚醒名はUIで見せ、台詞中では多用しない。

例:

```text
悪い例: もう迷いは消えた。私は完全な戦士だ。
良い例: 迷いは消えない。それでも、迷ったまま守る場所を選ぶ。
```

## 撃破後

撃破後は「理解されたが、改心していない」余韻を残します。

悪い例:

```text
私が間違っていた。今日から正義のために戦う。
```

良い例:

```text
お前の答えを、今は認めよう。だが俺の熱まで否定されたわけではない。
```

## カメラとUIを含めた台詞設計

台詞は画面演出とセットで書きます。

- 強い台詞の直前はBGMを薄くする。
- 観測片の表示時はUIノイズで「記録」を感じさせる。
- アルカナ関連の初期伏線は白い声、後ろ姿、記録音で出す。
- キャシヴァは暗いBGMを笑いで反転させる。
- メカゾ0アは台詞より命令ログで思想を出す。

## 禁止ワード感

アルカナやメーヴに「愚かな人間ども」「我が力にひれ伏せ」のような覇王語を使わない。  
キャリバーやアルディンに軽すぎる冗談を言わせない。  
カガチの粗さを、単なる乱暴や罵倒にしない。
