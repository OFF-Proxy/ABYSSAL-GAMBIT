## 0. 現状サマリ（2026-05-29 時点）

**実装済み（強い土台）**: Chapter 1（4ステージ×33ラウンド・中ボス/章ボス・コスト解放ゲート）、TFT式経済（基本5＋利子）、オーグメント80種（Silver30/Gold25/Prism25・HUD/ツールチップ/リロール）、シナジー＆実数値表示（全28体）、雑魚ユニット、各種UI、JA/EN ローカライズ、AUGMENT_FIXES / UI_POLISH 反映済み。

**リリースを阻む欠落（2026-05-29 改訂・ビジョン反映後）**:
1. **章をまたぐメタ進行が未実装** = 本作最大の差別化（過去章で倒したボスが仲間になり上位章へ）。現状ボス報酬は1ラン内のみ。MARKET_POSITIONING §2①。
2. **ボス前の掛け合い（会話演出）が未実装** = 東方風のストーリー性・ボスの個性（MARKET_POSITIONING §2②）。
3. **スコア/ランキング（E4）が未実装** + 永続化層が未着手。
4. **リリース固有の「ゲームの皮」が薄い** = タイトル/メニュー、初見導線、設定永続化、ビルド/パッケージング、Steam統合。
5. **バランス/プレイテスト未実施** = P6。

> **方針（改訂）**: 確定事項 = **Steam 買い切り / ソーシャルはランキング中心の軽い繋がり / アドベンチャー進行型を正面から作る**。
> 理想は「数十章」だが、**発売時に数十章すべては非現実的**。よって:
> - **メタ進行の "システム" は発売時に必須**（差別化の核なので後回し不可）。
> - **章は launch 時に少数（目安 3〜5 章）でループを成立させ**、以降アップデートで増やす（章追加が容易な構造は `BuildChapterRounds(chapter)` で既に着手済み）。
> - つまり「Chapter 1 だけ磨いて出す」ではなく「**メタ進行が回る最小の章数 + 収集ループ**を出す」がゴール。

## 4. リリース判定チェックリスト（製品版ゲート）

- [ ] launch章（3〜5章）を最初から最後まで、クラッシュ/詰みなく通しプレイできる
- [ ] **章ボスを倒すと仲間になり、次章の編成で使える（メタ進行が1周回る）**
- [ ] **ボス戦前に掛け合い（会話）が表示される**
- [ ] 章進捗・所持ボス仲間・ベストスコアが再起動後も保持される
- [ ] スコア/タイムが記録され、ランキング（最低ローカル）が表示される
- [ ] 設定（音量/言語/速度）が再起動後も保持される
- [ ] 初見プレイヤーが説明なしで基本操作に到達できる（チュートリアル or ガイド）
- [ ] JA/EN 両方でテキスト欠落・はみ出しがない
- [ ] productName/アイコン/バージョン/companyName が正式値
- [ ] Steam実績・クラウドセーブが動作（統合する場合）
- [ ] ストアページ（短文/詳細/タグ/スクショ5+/トレーラー）が揃っている
- [ ] 価格・リリース日・対応言語が設定済み
